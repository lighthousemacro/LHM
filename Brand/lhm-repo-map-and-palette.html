One page: canonical brand colors (signature palette + chart theme tokens) and how Git visibility relates to “open source” risk. Open this file locally in a browser; it is marked noindex if ever hosted as a static page.

Pillar docs & trading strategy (local only)

Policy: Twelve full pillar files (~1,200–1,750 lines each) and trading-strategy material stay under Strategy/ on your machine. They are in .gitignore and are not pushed to GitHub.

Ignored: Strategy/Pillar_*/, Strategy/md/, asset-class frameworks. Older commits may still have copies.

What is and is not on GitHub

“On GitHub” here means tracked by git and pushed. Files listed in .gitignore are normally not uploaded unless someone forced them in earlier (legacy tracked paths can remain until removed).

Area Typical state Sensitivity (if repo were public)
Data/databases/, Data/raw/, *.parquet, large CSV panels gitignored Low on GitHub — excluded by design; never commit master DB dumps.
.env, keys, API_KEYS.md gitignored Low on GitHub — still rotate any key ever leaked in history.
Strategy/Pillar_*/, Strategy/md/, trading masters gitignored Not pushed — full pillar docs and trading strategy stay local.
lighthouse_quant/ tracked Medium — model structure and code; rarely raw data.
Scripts/data_pipeline/ tracked Medium — operational logic; series IDs are not secrets; keys stay in env.
Brand/ tracked Low–medium — brand kit; large font bundles may be license-sensitive (check OFL/README in tree).
Archive/, External/, Working/ mixed often ignored for new files Review — many paths are still tracked historically; treat as cruft / third-party until cleaned.
Charts/*.png, Outputs/ gitignored Low — generated artifacts.
Verdict for a private institutional repo: Keeping code + methodology in git while databases, env secrets, and rendered outputs stay gitignored is a normal and defensible layout. If the remote were public, the main concern is Strategy/ and any lingering tracked material under Archive/ / External/ — not the pipeline skeleton alone.
